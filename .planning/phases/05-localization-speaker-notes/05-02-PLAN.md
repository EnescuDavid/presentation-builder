    Create `docs/speaker-notes.md` as the comprehensive speaker notes reference. This document covers:

    **1. Overview section** explaining that:
    - Speaker notes are optional and generated separately from slides (per D-07)
    - Notes live in a per-presentation notes file (per D-08)
    - At generation time, Claude reads the notes file and populates `<aside class="notes">` inside each `<section>` (per D-09)
    - reveal.js displays notes in speaker view (press `S` key) — `showNotes: false` is already configured in the skeleton

    **2. Notes File Format section** — use YAML format for structured data:
    ```yaml
    # Speaker Notes — [Presentation Name]
    # Generated for: [audience type]

    slides:
      - id: "title"
        component: "title"
        notes: |
          Willkommen zur Vorstandssitzung.
          Kernbotschaft: Die digitale Transformation zeigt messbare Ergebnisse.
          Zeitrahmen: 2 Minuten.

      - id: "marktanalyse"
        component: "text-heavy"
        notes: |
          Drei zentrale Erkenntnisse aus der Marktanalyse hervorheben.
          Auf Nachfragen zu Wettbewerbsdaten vorbereitet sein.

      - id: "metrics-q3"
        component: "metrics"
        notes: |
          Jede Kennzahl einzeln erklären.
          +23% Umsatzwachstum ist der Headline-Wert — hier Vergleich zum Vorjahr betonen.
    ```

    The `id` field matches the slide's `id` attribute in HTML (`<section id="title">`). If no `id` attribute exists, slides are matched by order (first slide entry = first `<section>`).

    **3. HTML Injection section** explaining the mechanism:
    - Claude reads the notes YAML file
    - For each slide entry, Claude adds an `<aside class="notes">` block inside the corresponding `<section>`:
      ```html
      <section id="title" data-master="hide" data-component="title">
        <div class="comp-title">
          <!-- slide content -->
        </div>
        <aside class="notes">
          Willkommen zur Vorstandssitzung.
          Kernbotschaft: Die digitale Transformation zeigt messbare Ergebnisse.
          Zeitrahmen: 2 Minuten.
        </aside>
      </section>
      ```
    - The `<aside class="notes">` MUST be a direct child of `<section>`, placed AFTER the component wrapper div
    - reveal.js automatically picks up these notes for speaker view
    - Notes support basic HTML: `<br>`, `<strong>`, `<em>`, `<ul>/<li>` for structured talking points

    **4. Audience-Aware Notes section** (per D-10):
    - Notes should adapt tone and detail based on the target audience
    - Table of audience types and note style:

    | Audience | Note Style | Detail Level | Emphasis |
    |----------|-----------|-------------|----------|
    | Vorstand (C-Suite) | Formal, concise | High-level, strategic | Business impact, ROI |
    | Fachbereich (Technical) | Precise, data-driven | Deep technical detail | Architecture, metrics |
    | Vertrieb (Sales) | Energetic, benefit-focused | Customer-oriented | Value proposition, cases |
    | Workshop | Interactive, question-prompting | Step-by-step guidance | Participation cues |
    | Intern (Internal) | Casual, transparent | Balanced | Progress, next steps |

    **5. Speaker View Usage section:**
    - Press `S` to open speaker view in a new window
    - Speaker view shows: current slide, next slide, notes, elapsed time
    - Notes are not visible to the audience — only in the speaker window
    - To toggle notes visibility on the main display: set `showNotes: true` in reveal.js config (useful for rehearsal)
