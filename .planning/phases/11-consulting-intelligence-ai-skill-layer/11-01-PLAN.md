Add an "Action title guidance" line to each of the 14+ component sections in component-catalog.md (per D-03, CONSULT-04). Insert after the existing "Required slots" or "Optional slots" line in each component section. Use German examples where possible per German-first convention.

**Per-component guidance (exact text to add):**

1. **Title:** `- **Action title guidance:** Hero text IS the action title -- state the core message or thesis. Good: "Cloud-Migration senkt Kosten um 30%". Bad: "Einleitung", "Willkommen".`

2. **Section Break:** `- **Action title guidance:** EXCEPTION -- use short topic labels (2-4 words), not action titles. This is a structural divider. Good: "Marktanalyse", "Naechste Schritte". Bad: full sentences.`

3. **Text-Heavy:** `- **Action title guidance:** State the argument or conclusion the text supports. Good: "Drei Faktoren treiben das Wachstum". Bad: "Hintergrund", "Details".`

4. **Two-Column:** `- **Action title guidance:** State the comparison insight or relationship. Good: "Kosten sinken, waehrend Qualitaet steigt". Bad: "Vergleich", "Uebersicht".`

5. **Metrics:** `- **Action title guidance:** State what the numbers prove, not what they are. Good: "Umsatz stieg um 15% durch APAC-Expansion". Bad: "Kennzahlen", "Umsatzuebersicht".`

6. **Image Full-Bleed:** `- **Action title guidance:** State the emotional or aspirational message. Good: "Unsere Vision fuer 2030". Bad: "Bild", "Impression".`

7. **Agenda:** `- **Action title guidance:** EXCEPTION -- use "Agenda" or "Tagesordnung". Structural slide, not content slide.`

8. **Summary:** `- **Action title guidance:** State the key takeaway. Good: "Drei Massnahmen sichern den Projekterfolg". Bad: "Zusammenfassung", "Fazit".`

9. **Contact:** `- **Action title guidance:** EXCEPTION -- use speaker name or "Kontakt". Structural slide.`

10. **Comparison:** `- **Action title guidance:** State the verdict or recommendation. Good: "Loesung B uebertrifft A in allen Kriterien". Bad: "IST vs. SOLL", "Vergleich".`

11. **Timeline:** `- **Action title guidance:** State the outcome or milestone. Good: "Migration abgeschlossen bis Q2 2027". Bad: "Zeitplan", "Roadmap".`

12. **Quote:** `- **Action title guidance:** State why the quote matters or what it proves. Good: "Kundenzufriedenheit bestaetigt den Ansatz". Bad: "Zitat", "Kundenstimme".`

13. **Card Grid:** `- **Action title guidance:** State what the collection demonstrates. Good: "Vier Services decken die gesamte Wertschoepfungskette ab". Bad: "Services", "Angebot".`

14. **Framework:** `- **Action title guidance:** State the insight the matrix reveals. Good: "Produkt A dominiert im Wachstumssegment". Bad: "BCG-Matrix", "Framework".`

Also add guidance to any v2.0 components (data-table, harvey-balls, sparkline, team, chart, mermaid, waterfall, code-block) following the same pattern: state the insight, not the component type.

For v2.0 data viz components, general pattern: `- **Action title guidance:** State the data insight, not the visualization type. Good: "[specific insight]". Bad: "[component name]".`
